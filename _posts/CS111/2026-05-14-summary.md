<div class="cat-label">Object-Oriented Programming</div>

<details class="note-card">
<summary>Writing Classes <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Build at least two custom character classes that extend engine base classes.</p>
<p><span class="lbl">Assessment</span> Code review across <code>Player.js</code>, <code>NPC.js</code>, <code>Enemy.js</code>.</p>
<p><span class="lbl">How met</span> <code>Gem extends Coin</code> in Gem.js and <code>Guard extends Enemy</code> in Guard.js — two fully custom character classes layered on top of the engine's base classes.</p>
</div>
</details>

<details class="note-card">
<summary>Methods &amp; Parameters <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Methods that take meaningful parameters (e.g. <code>collisionHandler(other, direction)</code>).</p>
<p><span class="lbl">Assessment</span> Code review for method signatures with 2+ parameters.</p>
<p><span class="lbl">How met</span> <code>handleCollisionEvent()</code> in Guard.js; <code>draw()</code>, <code>collect()</code>, <code>update()</code> in Gem.js each carry scoped logic; level constructors accept <code>(gameEnv)</code> as a parameter.</p>
</div>
</details>

<details class="note-card">
<summary>Instantiation &amp; Objects <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Instantiate game objects through the GameLevel configuration.</p>
<p><span class="lbl">Assessment</span> Code review of GameLevel setup objects.</p>
<p><span class="lbl">How met</span> <code>this.classes = [...]</code> in HeistL1.js, HeistL2.js, and HeistL3.js instantiates Player, Gem, Barrier, and Guard objects with config data.</p>
</div>
</details>

<details class="note-card">
<summary>Inheritance <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Class hierarchy at least 2 levels deep (e.g. GameObject → Character → Player).</p>
<p><span class="lbl">Assessment</span> Code review of the <code>extends</code> keyword and chain.</p>
<p><span class="lbl">How met</span> <code>Gem extends Coin</code> and <code>Guard extends Enemy</code> both build on engine base classes, creating a 2+ level hierarchy (e.g. GameObject → Enemy → Guard).</p>
</div>
</details>

<details class="note-card">
<summary>Method Overriding <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Override parent methods such as <code>update()</code>, <code>draw()</code>, <code>handleCollision()</code>.</p>
<p><span class="lbl">Assessment</span> Code review for polymorphic implementations.</p>
<p><span class="lbl">How met</span> Guard.js overrides <code>update()</code>, <code>stayWithinCanvas()</code>, and <code>handleCollisionEvent()</code>. Gem.js overrides <code>draw()</code>, <code>collect()</code>, and <code>update()</code>.</p>
</div>
</details>

<details class="note-card">
<summary>Constructor Chaining <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Use <code>super()</code> to chain parent constructors.</p>
<p><span class="lbl">Assessment</span> Code review for <code>super(data, gameEnv)</code> calls.</p>
<p><span class="lbl">How met</span> Gem.js calls <code>super(gemData, gameEnv)</code> after merging a data object. Guard.js calls <code>super(data, gameEnv)</code> then sets <code>this.velocity.y = -3</code>.</p>
</div>
</details>

<div class="cat-label">Control Structures</div>

<details class="note-card">
<summary>Iteration <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Loops for iterating over game object arrays and animation frames.</p>
<p><span class="lbl">Assessment</span> Code review for <code>for</code>, <code>forEach</code>, <code>while</code>.</p>
<p><span class="lbl">How met</span> heistMusic.js uses <code>.filter()</code> across API track results (<code>tracks.filter(...)</code>, <code>candidates.filter(...)</code>). Level files iterate <code>this.classes</code> every frame.</p>
</div>
</details>

<details class="note-card">
<summary>Conditionals <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Collision detection and state transitions driven by conditions.</p>
<p><span class="lbl">Assessment</span> Code review for <code>if/else</code> and nested conditions.</p>
<p><span class="lbl">How met</span> Guard.js: <code>if (!this.playerDestroyed &amp;&amp; this.collisionChecks())</code>. Gem.js: <code>if (this.permanentlyCollected) return</code>. heistMusic.js: <code>if (!this.started)</code> / <code>else if (this.isPlaying)</code>.</p>
</div>
</details>

<details class="note-card">
<summary>Nested Conditions <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Multi-layer game logic (e.g. power-up + collision + direction).</p>
<p><span class="lbl">Assessment</span> Code review for multi-level conditionals.</p>
<p><span class="lbl">How met</span> Guard.js <code>stayWithinCanvas()</code> has four nested boundary checks with velocity reversal. heistMusic.js <code>toggleMusic()</code> has nested <code>if / else if / else</code> for play state.</p>
</div>
</details>

<div class="cat-label">Data Types</div>

<details class="note-card">
<summary>Numbers <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Numeric properties for position, velocity, score tracking.</p>
<p><span class="lbl">Assessment</span> Code review for numeric values.</p>
<p><span class="lbl">How met</span> <code>velocity.y = -3</code>, <code>value: 5</code>, <code>SCALE_FACTOR: 10</code>, <code>STEP_FACTOR: 1000</code>, <code>volume = 0.3</code>, and canvas math like <code>width * 0.1406</code> across the level files.</p>
</div>
</details>

<details class="note-card">
<summary>Strings <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Character names, sprite paths, and game state labels.</p>
<p><span class="lbl">Assessment</span> Code review for string handling.</p>
<p><span class="lbl">How met</span> <code>id: 'gem'</code>, <code>name: 'mainplayer'</code>, <code>src: path + "/images/projects/heist-exe/heist-mc.png"</code>, plus iTunes query strings in heistMusic.js.</p>
</div>
</details>

<details class="note-card">
<summary>Booleans <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> State flags like <code>isJumping</code>, <code>isPaused</code>, <code>isVulnerable</code>.</p>
<p><span class="lbl">Assessment</span> Code review for boolean logic.</p>
<p><span class="lbl">How met</span> <code>permanentlyCollected: false</code>, <code>isHostile: true</code> (Guard), <code>this.started</code>, <code>this.isPlaying</code>, and <code>this.userActivated</code> all used as runtime state flags in Gem.js and heistMusic.js.</p>
</div>
</details>

<details class="note-card">
<summary>Arrays <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Collections of game objects and level data.</p>
<p><span class="lbl">Assessment</span> Code review for array operations.</p>
<p><span class="lbl">How met</span> <code>this.classes = [...]</code> in all three level files. <code>queries</code>, <code>candidates</code>, and <code>pool</code> arrays in heistMusic.js drive track selection.</p>
</div>
</details>

<details class="note-card">
<summary>JSON Objects <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Configuration objects with sprite data.</p>
<p><span class="lbl">Assessment</span> Code review for object literals.</p>
<p><span class="lbl">How met</span> Sprite config objects like <code>sprite_data_mc</code>, <code>gem_data_1</code>, and <code>border_top</code> are JSON objects with nested properties such as <code>hitbox: { widthPercentage: 0.45 }</code>.</p>
</div>
</details>

<div class="cat-label">Operators</div>

<details class="note-card">
<summary>Mathematical Operators <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Physics calculations — gravity, velocity, collision math.</p>
<p><span class="lbl">Assessment</span> Code review for <code>+ - * /</code> in physics code.</p>
<p><span class="lbl">How met</span> <code>this.position.y += this.velocity.y</code>, <code>this.velocity.y *= -1</code> (bounce), <code>Math.min(canvas.width, canvas.height) / 3</code>, <code>width * 0.55</code> for proportional positioning.</p>
</div>
</details>

<details class="note-card">
<summary>String Operations <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Path concatenation and dynamic text display.</p>
<p><span class="lbl">Assessment</span> Code review for template literals and concatenation.</p>
<p><span class="lbl">How met</span> <code>path + "/images/projects/heist-exe/gem.png"</code> (concatenation) and <code>`Gem collected! +${this.value} | Total: ${this.gameEnv?.stats?.coinsCollected}`</code> (template literal) in Gem.js.</p>
</div>
</details>

<details class="note-card">
<summary>Boolean Expressions <span class="arrow">▶</span></summary>
<div class="note-body">
<p><span class="lbl">Evidence</span> Compound conditions in game logic.</p>
<p><span class="lbl">Assessment</span> Code review for <code>&amp;&amp;</code>, <code>||</code>, <code>!</code>.</p>
<p><span class="lbl">How met</span> <code>!this.playerDestroyed &amp;&amp; this.collisionChecks()</code> in Guard.js; <code>!VOCAL_HINTS.test(name)</code> and <code>candidates.length &gt; 0 ? candidates : pool</code> in heistMusic.js.</p>
</div>
</details>
