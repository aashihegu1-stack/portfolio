## What I Learned in CS 111

This class was my full introduction to programming — from the very first `print("Hello")` to building small projects with classes, files, and tests.

## The Big Topics

- **[Object Oriented Programming]({{site.baseurl}}/cs111/oop)** — modeling things as classes with data and behavior.
- **[Control Structures]({{site.baseurl}}/cs111/control-structures)** — `if`, `for`, and `while` for decisions and repetition.
- **[Data Types]({{site.baseurl}}/cs111/data-types)** — picking the right container for each kind of value.
- **[Operators]({{site.baseurl}}/cs111/operators)** — arithmetic, comparison, and logic.
- **[Input/Output]({{site.baseurl}}/cs111/input-output)** — talking to users and files.
- **[Documentation]({{site.baseurl}}/cs111/documentation)** — explaining code to future me.
- **[Debugging]({{site.baseurl}}/cs111/debugging)** — finding and fixing problems methodically.
- **[Testing &amp; Verification]({{site.baseurl}}/cs111/testing)** — proving the code actually works.

## How It All Fits Together

A real program ties every single one of these together:

1. **Data types** hold the information.
2. **Operators** combine and compare values.
3. **Control structures** decide what to do.
4. **OOP** organizes the data and logic into reusable pieces.
5. **I/O** moves information in and out.
6. **Documentation** explains what's going on.
7. **Debugging** is how you survive when things break.
8. **Testing** is how you make sure it stays working.

## What's Next

CS 111 is the foundation. From here I can keep going into algorithms, web development, data science, or any other branch of computing — and the habits I built in this class (clear code, careful testing, good documentation) will carry forward into all of them.
