## The Five Concepts

<div class="concept-grid">

<details class="oop-card">
<summary>1. Writing Classes</summary>
<p>A class is a blueprint or template used to create objects. It defines the initial state (properties like health or position) and the behaviors (methods like move or attack) that all objects created from that blueprint will have. It allows you to organize your code into logical, reusable structures.</p>
</details>

<details class="oop-card">
<summary>2. Methods &amp; Parameters</summary>
<p>Methods are functions that live inside a class and represent the actions an object can take. Parameters are the placeholders that allow you to pass specific information into those methods. For example, a <code>jump(height)</code> method uses the <code>height</code> parameter to determine exactly how high that specific jump should be.</p>
</details>

<details class="oop-card">
<summary>3. Instantiation &amp; Objects</summary>
<p>Instantiation is the act of creating a living "copy" of a class in your game's memory. While the class is just the blueprint, the Object is the actual house built from it. You can instantiate five different "Enemy" objects from one "Enemy" class, and each can have its own unique health and position during gameplay.</p>
</details>

<details class="oop-card">
<summary>4. Method Overriding</summary>
<p>This occurs when a Child class provides a specific implementation of a method that is already defined in its Parent class. For example, if the Parent <code>Enemy</code> has a generic <code>attack()</code> method, the Child <code>Boss</code> can "override" it to perform a much more powerful special attack instead of the default one.</p>
</details>

<details class="oop-card">
<summary>5. Constructor Chaining</summary>
<p>The constructor is a special method that runs automatically when an object is created. Chaining involves using the <code>super()</code> keyword to ensure that the Parent's constructor runs before the Child's constructor logic. This ensures the object is properly set up with all basic "Parent" traits before the "Child" adds its unique features.</p>
</details>

<div class="tab-panel active" data-panel="enc">
<h3>Encapsulation</h3>
<p>Imagine each object like a backpack — everything it needs is zipped inside, and only the owner decides what to pull out. A class keeps its data and methods together as one self-contained unit, so the rest of my code can't reach in and break things by accident.</p>

```javascript
this.type = "Guard";
this.isHostile = true;
```
</div>

<div class="tab-panel" data-panel="inh">
<h3>Inheritance</h3>
<p>Inheritance is how a new class gets a head start. When I write <code>class Pirate extends Character</code>, my Pirate already knows how to move, fall, and render — it inherited all of that from Character. I only have to write what makes a Pirate <em>different</em>.</p>

```javascript
class Guard extends Character { ... }
```
</div>

<div class="tab-panel" data-panel="poly">
<h3>Polymorphism</h3>
<p>Polymorphism is when many classes answer to the same method name but each responds in its own way. Every character has a <code>handleCollision()</code>, but a Pirate handles collisions like a Pirate would — by getting hostile when the Player wanders too close.</p>

```javascript
handleCollision(other, direction) {
    if (other instanceof Player) {
        if (this.distanceTo(other) < 50) {
            this.reaction("hostile");
        }
    }
}
```
</div>

<div class="tab-panel" data-panel="abs">
<h3>Abstraction</h3>
<p>Abstraction is the "I don't need to know how, I just need it to work" pillar. When I call <code>super.update()</code>, all of the parent's physics, animation, and movement logic runs in one line — and I don't have to think about any of it. I just add what's unique to my class on top.</p>

```javascript
update() {
    super.update();        // parent logic runs here
    this.checkProximity(); // pirate-specific behavior
}
```
</div>

(function() {
    // ===== Hierarchy =====
    const nodeInfo = {
        GameObject: {
            title: "GameObject",
            props: ["+position", "+draw()"],
            blurb: "The root of every visible thing in the game. If it appears on screen, it starts here."
        },
        Character: {
            title: "Character extends GameObject",
            props: ["+movement", "+gravity", "+update()"],
            blurb: "Adds physics and per-frame updates on top of GameObject."
        },
        Player: {
            title: "Player extends Character",
            props: ["+handleKeyInput()"],
            blurb: "A Character that listens to the keyboard. Inherits everything above."
        },
        Guard: {
            title: "Guard extends Character",
            props: ["+isHostile = true", "+handleCollision()", "+checkProximity()"],
            blurb: "A Character that reacts to the Player. Inherits everything above."
        }
    };

    const info = document.getElementById('hier-info');
    document.querySelectorAll('.hier-node').forEach(n => {
        n.addEventListener('click', () => {
            document.querySelectorAll('.hier-node').forEach(x => x.classList.remove('active','ancestor'));
            document.querySelectorAll('.hier-line').forEach(x => x.classList.remove('lit'));
            document.querySelectorAll('.hier-row').forEach(x => x.classList.remove('lit'));

            n.classList.add('active');
            const parents = (n.dataset.parents || '').split(',').filter(Boolean);
            parents.forEach(p => {
                document.querySelector(`.hier-node[data-node="${p}"]`)?.classList.add('ancestor');
            });

            // Light the connecting lines
            const name = n.dataset.node;
            if (name === 'Character' || name === 'Player' || name === 'Guard') {
                document.querySelector('[data-line="GameObject-Character"]')?.classList.add('lit');
            }
            if (name === 'Player') {
                document.querySelector('[data-line="Character-Player"]')?.classList.add('lit');
            }
            if (name === 'Guard') {
                document.querySelector('[data-line="Character-Guard"]')?.classList.add('lit');
            }

            const d = nodeInfo[name];
            info.innerHTML = `
                <h4>${d.title}</h4>
                <p style="color:#ddd;font-size:0.92em;margin:6px 0 10px 0;">${d.blurb}</p>
                <ul>${d.props.map(p => `<li>${p}</li>`).join('')}</ul>
            `;
        });
    });

    // ===== Tabs =====
    document.querySelectorAll('.tab-btn').forEach(btn => {
        btn.addEventListener('click', () => {
            document.querySelectorAll('.tab-btn').forEach(b => b.classList.remove('active'));
            document.querySelectorAll('.tab-panel').forEach(p => p.classList.remove('active'));
            btn.classList.add('active');
            document.querySelector(`.tab-panel[data-panel="${btn.dataset.tab}"]`)?.classList.add('active');
        });
    });

    // ===== Quiz =====
    document.querySelectorAll('.q-block').forEach(block => {
        const correct = block.dataset.correct;
        const feedback = block.querySelector('.q-feedback');
        block.querySelectorAll('.q-opt').forEach(opt => {
            opt.addEventListener('click', () => {
                block.querySelectorAll('.q-opt').forEach(o => {
                    o.classList.remove('correct','wrong');
                    o.disabled = true;
                });
                if (opt.dataset.opt === correct) {
                    opt.classList.add('correct');
                    feedback.textContent = "Correct!";
                    feedback.className = "q-feedback show right";
                } else {
                    opt.classList.add('wrong');
                    block.querySelector(`.q-opt[data-opt="${correct}"]`).classList.add('correct');
                    feedback.textContent = "Not quite — the highlighted answer is the right one.";
                    feedback.className = "q-feedback show nope";
                }
            });
        });
    });
})();
</script>
